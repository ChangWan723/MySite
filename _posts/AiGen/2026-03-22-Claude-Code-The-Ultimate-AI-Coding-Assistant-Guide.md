## Getting Started with Claude Code

### Installation
Claude Code can be installed through multiple methods:

```bash
# Via npm (recommended for terminal usage)
npm install -g @anthropic-ai/claude-code

# Via Homebrew (macOS)
brew install claude-code

# Direct download from Anthropic's website
# Visit https://claude.ai/code for platform-specific installers
```

### Basic Configuration
After installation, configure Claude Code for your environment:

```bash
# Initialize configuration
claude config init

# Set default model (Claude 3.5 Sonnet is recommended for balance)
claude config set default_model claude-3-5-sonnet-20241022

# Configure project context
echo "Project: MyAwesomeApp" > CLAUDE.md
echo "Tech Stack: React, Node.js, PostgreSQL" >> CLAUDE.md
```

### Your First Claude Code Session
Start a simple session to test the setup:

```bash
# Start an interactive session
claude

# Or run a one-off command
claude -p "Write a Python function to calculate Fibonacci numbers"
```

```markdown
- Build: `npm run build`
- Lint: `npm run lint`
```

### 2. Custom Slash Commands
Create reusable workflows with custom commands:

```bash
# Define a PR review command
claude config add-command review-pr "
  # Review the current PR
  1. Fetch latest changes: git fetch origin
  2. Check diff: git diff origin/main..HEAD
  3. Analyze code quality and potential issues
  4. Suggest improvements
"

# Use the command
/review-pr
```

### 3. Multi-Agent Orchestration
For complex tasks, use Agent Teams:

```bash
# Start a team for a full-stack feature
claude team create "Implement user authentication" \
  --roles "frontend,auth-backend,database,security" \
  --size 4
```

### 4. Hooks and Automation
Set up automation with pre/post hooks:

```bash
# Auto-format code after every edit
claude config add-hook post-edit "prettier --write {file}"

# Run tests before committing
claude config add-hook pre-commit "npm test"
```
